1. REVISED PROJECT GOAL & CORE PARADIGM
--------------------------------------------------------------------------------
The primary objective of x2gAI-OS is to deploy a high-performance, deterministic,
local-first agentic operating system that manages workflows, schedules routines,
and runs pipelines across your entire development, enterprise, research, and 
web application portfolio.[cite: 1, 2]

To optimize the developer experience, performance footprint, and enterprise layout 
flexibility, the architectural interface paradigm is shifted to a dual-plane UI:

* PRIMARY INTERFACE: An enterprise-grade Web Application Dashboard & Command Center.
  This web panel serves as the live operational deck, running natively inside a
  Docker container on the host machine. The front-end is built using Angular, 
  leveraging DevExtreme controls to deliver high-performance grids, rich multi-tab 
  layouts, complex menus, and analytical charts.
* SECONDARY INTERFACE: Multi-Vault Obsidian Architecture.
  Your existing Obsidian vaults serve as an externalized reactive database, long-term
  semantic memory cache ("2nd Brain"), and secondary control configuration plane.[cite: 1, 2]
  The daemon monitors updates through file-system tracking.[cite: 1, 2]

All agent orchestrations, loop routing, and state definitions are declared using
strict JSON configuration schemas, discarding brittle YAML or markdown frontmatter
for state-machine tracking.[cite: 1]

2. DETAILED STRUCTURAL COMPONENT REQUIREMENT
--------------------------------------------------------------------------------
[1] Local Ollama Pipeline
    * Primary local execution point using quantized weights on local hardware.[cite: 1]
    * Communicates via local REST endpoints on http://localhost:11434.[cite: 1]
    * Fallback mechanics shift workloads to commercial APIs (Anthropic Claude,
      OpenAI, Gemini) only when required context lengths exceed local VRAM capabilities 
      or during verified local latency bottlenecks.[cite: 1]

[2] Web Application & Docker Ecosystem
    * Containerized deployment hosting the main user dashboard, live agent execution logs,
      and centralized telemetry graphs.[cite: 1]
    * Built out via an Angular Single Page Application (SPA) serving a clean, lightweight, 
      modular component layout populated with rich DevExtreme controls.
    * Exposes an internal WebSocket connection to the Node.js daemon for instantaneous,
      zero-polling telemetry streaming and dynamic file-tree discovery.[cite: 1]

[3] Secondary Interface (Obsidian Multi-Vault Integration)
    * The system interfaces natively with your existing, pre-configured Obsidian Vaults.[cite: 1]
    * The Fable daemon uses asynchronous file system trackers to safely read/write files 
      across isolated vaults without corrupting local markdown histories or link graphs.[cite: 1]

[4] DevExtreme Command Center & Multi-Module Dashboard
    * Features a feature-rich visual hub showcasing multi-view layouts, expandable left 
      side-navigation menus, data grids, and task boards.
    * Modules are fully dynamic: users can instantly load up separate panels for Web Apps,
      Digital Pathology workloads, ML training logs, and Logistics tickets.
    * Includes an active Command Center terminal component allowing the manual input of
      direct text syscalls, routine injections, or manual overrides.[cite: 1]

[5] Paperclip-Style State & Token Governance
    * Enforces absolute execution determinism. Dynamic loops or unconstrained autonomous
      paths are prohibited; state modifications use definitive F# Discriminated Unions.[cite: 1]
    * Every transaction is passed through an Interceptor Pipeline that evaluates a local
      append-only token ledger.[cite: 1] If an agent hits a preset commercial budget ceiling, the loop
      aborts safely and transfers context flags back to the local Ollama instance.[cite: 1]

[6] Pure JSON-Based Configuration Architecture
    * System settings, agent profile structures, skills lists, and pipeline dependencies
      are structured entirely using rigid JSON schemas.[cite: 1]
    * This guarantees rapid native parsing by Node.js, simple schema validation via standard tools,
      and seamless serialization with the Fable 5 data model.[cite: 1]

3. DYNAMIC ENVIRONMENT UNDERSTANDING & AUTONOMOUS TREE DISCOVERY
--------------------------------------------------------------------------------
The background Fable daemon executes automated structural scanning routines. It does 
not rely on statically defined project structures; it proactively discovers your ecosystem by:
* Mapping Local Trees: Recursively walking through `~/.claude`, `X2G_infrastructure`, 
  and `X2GVentures` to read directory topologies.
* Context Verification: Locating and indexing individual project `.claude.md` or 
  `.claude` files inside every sub-folder to glean localized boundaries, rules, and scopes.
* Dynamic Scaffolding & On-the-Fly Generation: Acting as a meta-generator. From within
  the DevExtreme dashboard interface, you can issue commands to quickly spin up brand 
  new modular sub-projects, micro-apps, or template new websites. The system automatically
  injects the new JSON schema manifests, populates standard configurations, writes a fresh 
  localized `.claude.md` file, and appends a new module item to your Angular side-navigation menu.

5. PORTFOLIO & PROJECT ALIGNMENT MATRIX
--------------------------------------------------------------------------------
The x2gAI-OS core engine is natively adapted to automate your highest frequency
workflows and production domains:

* X-TMS (Transportation Management System) Automation:
  The daemon reads inbound freight alerts, parses shipping itineraries, and processes
  rate confirmation paperwork.[cite: 1] It interfaces with external infrastructure via type-safe
  wrappers, managing dispatch alerts using approved Twilio 10DLC routes.[cite: 1]
  
* TrialLens & Clinical Research Workflows:
  Integrates complex medical informatics parsing schemas.[cite: 1] It automates semantic processing
  of clinical test data, study protocol layouts, and research publications, using 
  alphanumeric-safe medical records parsing parameters.[cite: 1]

* Digital Pathology Workflows:
  Manages pipeline state tracking for microservice imaging layers, analyzing slide metadata,
  and orchestrating analytical steps deterministically.

* ML Pipeline & Infrastructure Monitoring:
  Exposes model validation metrics, checkpoints, and parameter states directly to 
  DevExtreme charts and grids on the primary dashboard interface.

* Web Application Scaffold-on-Demand:
  Monitors existing web codebases via local folder scanners. Provides an interactive UI 
  form to generate new website directories, configure tracking structures, and append 
  functional endpoints seamlessly.

* Personal Knowledge Graph Optimization:
  Operates on your existing Obsidian vaults to structure raw text caches, process dev logs, 
  and generate structured, high-efficiency system tracking summaries without creating 
  redundant file trees or processing loops.[cite: 1]
